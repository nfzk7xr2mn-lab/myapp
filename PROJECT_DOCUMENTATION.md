## 1. Overview

A personal **browser dashboard** as a local web app (no framework, no build step), running on a Windows PC and displaying:

- **Default mode** (standard): A compact bar with clock, current appointment, mail status and day phase
- **Expert mode**: Full view with 4 tiles (Calendar, Focus, Actions, Mails) -- toggle with key `E`

**Technology stack:**
- Frontend: plain HTML/CSS/JavaScript (no framework)
- Backend for write access: Node.js HTTP server (port 9001)
- Data pipeline: PowerShell scripts via Windows Task Scheduler
- Data formats: `.ics`, `.json`
- Development server: `npx live-server` (port 5500)

## 3. Layout and Design

### 3.1 Modes: Default Mode and Expert Mode

The dashboard has two modes, toggled via key `E` or button in the top right. The mode is saved in `localStorage`. Default: Default mode (`expertMode = localStorage.getItem('expertMode') === '1'`).

The dashboard runs in **Chrome App mode** (`--app` flag), which enables real window resizing via JavaScript. `start-servers.ps1` opens Chrome automatically.

#### Default Mode (`mode-simple`)

Narrow bar at the bottom of the screen (64px high, full width):
```
[HH:MM]  [Weekday, Date]  [Phase]  |  [Calendar 2-line] [Reload]  |  [Mail N]  [Badges]  [Mode button]
```
- `#header` and `#tiles` are hidden
- Window is resized to 64px height and positioned at the bottom of the screen (`window.resizeTo` + `window.moveTo`)
- **Calendar block** (`#sb-cal`): Two-line column layout (flexbox column). Line 1 (`.sb-line1`): time first, current or next appointment. Line 2 (`.sb-line2`, smaller/dimmed): next appointment and/or free status. Example: Line 1 "bis 18:00 Teammeeting", Line 2 "ab 19:10 Tanzen, frei ab 20:00". Current appointment gold (`.sb-current`), next dimmed (`.sb-next`). Free status (`.sb-free-tag`): "jetzt frei" when no appointment is running, otherwise "frei ab HH:MM". Consecutive appointments (gap <= 5 min) count as continuously busy. `max-width: 55%`. Next to it a reload button (`#btn-refresh-cal`) for manually reloading ICS files (calls `loadICSAuto()`, spin animation as feedback).
- **Mails** (`#sb-mail`): Count of today's received mails (number only, no sender/subject). `min-width: 120px` guarantees visibility.
- **Badges** (`#sb-badges`): Compact counters right of the mail section. Shows count of unanswered direct/boss mails (gold pill) and open actions (green pill). Calculation: A direct mail counts as answered if a sent mail to the same sender exists (same or later day).
- Updates every minute

#### Expert Mode (`mode-expert`)

Full 4-tile view. `#simple-bar` is hidden. Window is resized to working area size (`screen.availWidth x screen.availHeight`) and positioned top-left. In the header right: Server status dot + mode toggle button (`#btn-mode-toggle-expert`, class `.btn-mode`) to switch back to default mode.

| `renderHeader()` | Calls `renderSimpleBar()` when `!expertMode` |
| `loadMails()` | Calls `renderSimpleBar()` when `!expertMode` |
| `loadICSAuto()` | Calls `renderSimpleBar()` when `!expertMode` |

## 4. Tile 1: Calendar

### Display
- Today's appointments with time and title
- Current appointment: gold highlighted (`cal-current`)
- Past appointments: gray, opacity 0.4 (`cal-past`)
- Tentative: dimmed, badge `?` (`cal-tentative`)
- Optional: dimmed, badge `opt` (`cal-optional`)
- After 17:00: preview of tomorrow's appointments (`cal-tomorrow`, opacity 0.55)

### ICS File: `data/calendar_YYYYMMDD.ics`
- Generated by `export_outlook_today.ps1`
- Loaded at startup and every 30 minutes automatically (`loadICSAuto`)

### ICS Parser (`parseICS(text, forDate)`)
- Reads `DTSTART`, `DTEND`, `SUMMARY`, `TRANSP`, `X-MICROSOFT-CDO-BUSYSTATUS`, `X-MYAPP-ROLE`
- `TRANSP:TRANSPARENT` -> not displayed (free appointments)
- `VALUE=DATE` without `T` -> all-day (`allDay: true`)
- UTC with `Z` suffix -> correctly converted to local time

### Sync Preparation
- `data/sync_files.json` is loaded at startup
- Click on appointment link opens `#modal-sync` with mail list
- Title in modal: `shortName(name)` -- shows only first name + first letter of last name

## 5. Tile 2: Focus (7 Tabs)

### Default Tab
- Saved in `localStorage` (`fokusTab`) -- persists after reload
- First visit without saved value: before 8am -> Sport, 8am-5pm -> Notizen, after 5pm -> Wissen

### Tab Notizen
- Quick notes from `data/notes.json`
- Collapsible via `<details>`, newest first
- Emoji lines in gold (`--gold`), blank line before each emoji line (from 2nd line onward); rest dimmed
- **`+ Notiz` button** opens modal
- **`Planung` button** triggers `plan_week.ps1` on-demand:
  - POST `/run-plan-week` -> write-server starts PS script via `execFile`
  - Polls every 8 seconds on `data/notes.json` until entry with `titel.startsWith('Planung KW')` and `ts >= triggeredAt` (max 8 min)
  - Double-click protection via `planPolling` flag

Data format `data/notes.json`:
```json
[{"titel":"Planung KW17-MO","datum":"21.04.","ts":1745276400000,"text":"..."}]
```
`ts` is Unix milliseconds (UTC), set by `plan_week.ps1`.

### Tab Sport
- Curated permanent list from `knowledge/sport.json`

### Tab Fuehrung / Tab Tech
- Flat list from `knowledge/learningplan.json`
- Checkbox progress in `knowledge/learningplan_progress.json`

### Tab Links
- `knowledge/links.json`: `[{"label":"...","url":"..."}]`

### Tab Wissen
- Audiobooks, postmortem, performance decline
- Collapsible: Title -> Summary -> Key insights

### Tab Netzwerk (Network)
- Auto-generated contact list from email correspondence and calendar events
- Sorted by interaction frequency (`freq` = `mailFreq + meetFreq`), switchable to alphabetical (A-Z)
- Per contact: Name (shortName format), editable role/notes, frequency badge, date range
- Click opens `#modal-netzwerk` to edit role
- Sort toggle (`Haeufigkeit` / `A-Z` / `Abteilung`) above the list, selection saved in `localStorage`
- **Search field**: Real-time filter above the sort bar, filters by name and role. Search text and cursor position are preserved on re-render.
- **Manual adding**: `+ Kontakt` button in the footer opens `#modal-add-contact`. Entered name is converted to shortName format via `toShortName()` (JS equivalent of PowerShell's `ConvertTo-ShortName`). Duplicate check by name.
- Data: `data/contacts.json`

Data format `data/contacts.json`:
```json
[{"name":"Daniel K.","rolle":"Tech Lead","first":"13.01.","last":"22.04.","mailFreq":12,"meetFreq":8,"freq":20,"_weekId":"2026-W17"}]
```
- `name`: shortName format (first name + first letter of last name), NO full last name
- `rolle`: Free text, user-editable in the dashboard
- `mailFreq`/`meetFreq`: Separate counters for mail correspondence and meeting participation
- `freq`: Sum, sort criterion
- `_weekId`: Internal week identifier for correct accumulation (prevents double-counting)

Contact extraction (in `export_outlook_mails.ps1`):
- Received mails: sender name -> shortName -> mailFreq++
- Sent mails: recipients (`;`-separated) -> each shortName -> mailFreq++
- Calendar: ATTENDEE CN values from `data/calendar_*.ics` of the week -> meetFreq++
- Self-filter: Own names ($MyDisplayNames) are skipped
- DL/SAP/Cloud/External filter: Names starting with "DL ", "SAP " or "Cloud ", containing "(external", or containing `_`/digits are skipped
- Department lookup: When `rolle` is empty or just "Mitarbeiter", the department is auto-appended from Outlook (Exchange `Resolve-Contact` -> `GetExchangeUser().Department`). Results are cached (`$deptCache`). Full names are only kept in memory, never persisted in `contacts.json`.
- Merge: `rolle` is never overwritten, frequencies are accumulated per week. Just before writing, `contacts.json` is re-read -- only roles that changed since the initial read (= user edits during export) are adopted.

## 9. PowerShell Data Pipeline

### 9.1 `secrets.ps1` (not committed)

Contains: `$ApiKey`, `$ProxyUri`, `$MyAddresses`, `$MyDisplayNames`, `$ChefName`, `$ChefEmail`, `$ExcludeSenders`, `$Mitarbeiter`.

**Never** put names, email addresses or API keys in committed scripts.

### 9.2 `export_outlook_today.ps1`
- Exports today and tomorrow as `data/calendar_YYYYMMDD.ics` each
- Free (`BusyStatus=0`) -> `TRANSP:TRANSPARENT`
- Tentative (`BusyStatus=1`) -> `X-MICROSOFT-CDO-BUSYSTATUS:TENTATIVE`
- Optional -> `X-MYAPP-ROLE:OPT-PARTICIPANT`
- Meeting attendees as `ATTENDEE;CN=Name:MAILTO:unknown` lines (for network contact extraction)

### 9.3 `export_outlook_mails.ps1`
- Exports Inbox + Sent: current Monday through today
- Writes `data/mails_today.json`, `data/mail_sync_status.json`, `data/config.json`
- Extracts contacts from mails + calendar ICS files -> `data/contacts.json`
- Timestamp via `[DateTimeOffset]::UtcNow.ToUnixTimeMilliseconds()`

### 9.4 `export_sync_prep.ps1`
- Checks tomorrow's calendar for 1:1 syncs
- Writes `data/KW{n}-{FirstName}.json` and `data/sync_files.json`

### 9.5 `summarize_mails.ps1`
- Reads `data/mails_today.json`, sends to Claude API via corporate proxy
- Writes `data/summary_KW{n}.json`: `{"summary":"...","ts":<unix-ms>}`

### 9.6 `plan_week.ps1`
- Runs daily at 18:00 and on-demand via `POST /run-plan-week`
- Evaluates all 5 weekdays (MO-FR) of the current week, each as a separate note
- Per day: Reads that day's ICS + mails received BEFORE that day (knowledge from the evening before)
- 5 separate API calls (one per day), mails are loaded once, filtered per day by date
- Sends to Claude API -> evaluates how well the day is planned (assistant perspective)
- Writes as `Planung KWxx-MO`, `-DI`, `-MI`, `-DO`, `-FR` into `data/notes.json` (same title replaced)
- All notes get the same `ts` value (Unix-ms) for polling detection in the dashboard
- `notes.json` is written once at the end (not 5 times)
- Skipped when `C:\Temp\myapp-modal.lock` exists
- Started via `execFile` (not spawn) -- important for Corporate Windows
- Polling timeout in the dashboard: 8 minutes (5 API calls can take 30-50 sec)

**Prompt concept:** Assistant evaluates the day, not a day planner.
Format: Emoji + colon + one sentence, max 5 lines, max 120 words, no markdown bold.
Topics: Meeting density, preparation needed, lunch break, what's well planned, open action items.

### 9.7 `setup.ps1`
One-time setup -- can be run any number of times (idempotent):
1. Checks Node.js, npx, `secrets.ps1`, `data/`, `knowledge/`, `C:\Temp`
2. Registers all 7 tasks via `schtasks /create /it /ru %USERNAME% /f`
3. Starts write-server immediately via `Start-Process`

**Note:** `schtasks /tr` requires escaped quotes for paths with spaces. Helper function `New-TR` builds correct `\"...\"`-strings.

| Task | Schedule |
|---|---|
| MyApp Write Server | on logon |
| MyApp Mail Export | 07:30 daily |
| MyApp Mail Export 1230 | 12:30 daily |
| MyApp Sync Prep | 17:00 daily |
| MyApp Kalender Export | 06:00 daily |
| MyApp Kalender Export 1200 | 12:00 daily |
| MyApp Tagesplanung | 18:00 daily |

## 15. Known Design Decisions and Pitfalls

- **execFile vs spawn for plan_week**: `spawn` with `detached:true` does not work on Corporate Windows -- output is lost, script does not complete. `execFile` solves this.
- **PowerShell -- ASCII ONLY**: No em-dashes, umlauts or non-ASCII in PS scripts -- not even in comments. Causes silent parser errors when PS is started non-interactively via Node.js.
- **CRLF in PS scripts**: PS scripts must have Windows line endings (CRLF). Convert with `unix2dos` after writing via Claude tools.
- **POWERSHELL constant**: Absolute path `%SystemRoot%\System32\WindowsPowerShell\v1.0\powershell.exe` instead of `powershell.exe` -- PATH for Node processes on Corporate Windows can be restricted.
- **Outlook COM**: Requires running Outlook + interactive session. Register tasks with `schtasks /it`.
- **schtasks /tr quoting**: For paths with spaces, inner quotes must be escaped. Helper function `New-TR` in `setup.ps1`.
- **Register-ScheduledTask blocked**: Corporate GPO prevents the PS cmdlet for normal users -> `schtasks.exe /create` as alternative.
- **UTC timestamp**: Use `[DateTimeOffset]::UtcNow.ToUnixTimeMilliseconds()`. JS side: set `timeZone: 'Europe/Berlin'` in `toLocaleTimeString` explicitly.
- **modal.lock in `C:\Temp\`**: Not in `data/` -- live-server would otherwise trigger a page reload.
- **UTF-8 without BOM**: `New-Object System.Text.UTF8Encoding $false` for all JSON write operations.
- **Planning polling**: Checks `n.ts >= triggeredAt` -- not just whether a note exists, but whether it was written after the button click. Double-click protection via `planPolling` flag. Interval 8 seconds.
- **Multi-trigger via schtasks**: `schtasks` only supports one trigger per task -> Mail Export and Calendar Export each as two separate tasks.
- **$pid reserved**: `$pid` is a reserved PS variable (process ID). Use `$procId` as loop variable.
- **expertMode default**: `localStorage.getItem('expertMode') === '1'` -- default is Default mode, only explicit `'1'` switches to Expert mode. First visit starts in the compact Default mode (SimpleBar).
- **fokusTab in localStorage**: Active Focus tab is saved in `localStorage` -- persists after reload. Time-based default only applies on first visit.
- **shortName for all personal names**: Use `shortName()` everywhere in the UI -- returns `FirstName L.`. Also applies to sync modal titles.
- **Refresh throttling**: Server health check 5 min (not 30 sec), mails 30 min (not 5 min), ICS 30 min (not 15 min), mail export at app start only 07-13h. Drastically reduces PS process starts.
- **debug-check.js**: Referenced in `index.html` (`<script src="debug-check.js">`), but file does not exist. Causes a 404 on load, but is not functionally critical.
- **live-server --ignore**: `data,knowledge` are excluded from watch, so PS write operations don't trigger a browser reload.
- **restart-writeserver.bat**: Kills all `node.exe` processes and restarts both servers. Caution: also kills other Node processes. Prefer `start-servers.ps1` (only kills ports 5500+9001).
- **ISOWeek not available in PS 5.1**: `[System.Globalization.ISOWeek]` requires .NET 5+. Windows PowerShell 5.1 uses .NET Framework 4.x. Use `$ci.Calendar.GetWeekOfYear($date, FirstFourDayWeek, Monday)` instead -- this is ISO 8601 compliant.
