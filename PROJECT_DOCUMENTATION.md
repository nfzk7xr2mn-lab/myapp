# Your Day. Your Direction. Your Decisions.

Project Documentation. As of: April 23, 2026.

---

## 1. Overview

Personal **browser dashboard** as a local web app (no framework, no build step) on Windows.

| Component | Technology |
|---|---|
| Frontend | HTML / CSS / JavaScript (single page, no framework) |
| Write Server | Node.js HTTP on port 9001 (`write-server.js`) |
| Dev Server | `npx live-server` on port 5500 |
| Data Pipeline | PowerShell scripts, triggered via Task Scheduler + on-demand |
| Data Formats | `.ics` (calendar), `.json` (everything else) |
| Browser | Chrome app mode (`--app` flag), window resize via JS |

## 3. Three Modes

Toggleable via `E` key (cycles: Simple -> Expert -> Focus) or button. Stored in `localStorage` (`dashMode`). Default: basic mode.

### Basic Mode

Narrow bar (56px) at the bottom of the screen. No emoji, text only.

```
[Clock] [Date] [Phase] | [Calendar 2-line] [Reload] | [Badges] [Mode]
```

- **Calendar** (`#sb-cal`): Current event (gold) + next (dimmed) + free status. After 5 PM: first event of the next day with weekday (e.g. "Thursday from 09:00 ...") when no events remain today. Hidden when completely empty.
- **Badges** (`#sb-badges`): Four pill badges -- events (gold), mails (gold), actions (green), Jira (blue). Count + label. Hidden when 0.
- Uniform font size (0.70rem) in the calendar area, no emojis.
- Refreshes every minute.

### Expert Mode

Full 4-tile view (Data, Instructions, Actions, Mails). No tile emojis. Header with daily quote (italic gold), quote refresh, server status dot, mode button.

### Focus Mode

Two-column layout: left compact (Calendar, Mails, Actions), right large (Instructions).

```
[Calendar compact]  | [                ]
[Mails compact]     | [ Instructions   ]
[Actions]           | [                ]
```

- **Calendar** (max 130px): Only current + next event. After 5 PM: first event tomorrow.
- **Mails** (max 150px): Only today's mails (top 5) with priority marker, no tabs.
- **Actions**: Remaining height on the left.
- **Instructions**: Full height on the right, double width.
- Header same as expert mode.

### Day Phase Backgrounds

Morning 06-11:30 | Noon 11:30-13 | Afternoon 13-17 | Evening 17-19:30 | Late Evening 19:30-22:30 | Night 22:30-06

### Color Palette

`--gold: #C9A84C` | `--gold-light: #E8C97A` | `--text: #f0ead6` | `--text-muted: #8a7d60`

Tiles: semi-transparent + `backdrop-filter: blur(20px)`, gold border.

## 4. Data Tile (Calendar)

- Today's events with time. Current: gold. Past: gray. Tentative: `?`. Optional: `opt`.
- After 5 PM: next day preview (expert mode: all events; basic/focus mode: first event with weekday).
- Source: `data/calendar_YYYYMMDD.ics` (generated by `export_outlook_today.ps1`). Loaded at start + every 30 min.
- ICS parser: `DTSTART/DTEND/SUMMARY/TRANSP/X-MICROSOFT-CDO-BUSYSTATUS/X-MYAPP-ROLE`. Free events invisible. UTC correctly converted.
- Sync prep: `data/sync_files.json`. Click on event link opens modal with mail list. Sent mails labeled as "sent" (instead of recipient list).

## 5. Instructions Tile (7 Tabs)

Default tab by time of day: before 8 -> Knowledge, 8-17 -> Notes, after 17 -> Knowledge.

### Notes
- `data/notes.json`, collapsible, newest on top. Emoji lines gold, rest dimmed.
- Planning notes (KWxx-MO/TU/...) under shared tree node, short label per day, timestamp next to KW header.
- Buttons: `+ Note` (modal), `Planning` (POST `/run-plan-week`, polling 8s, max 8 min.)
- Polling compares note timestamps (before vs. after), not browser time.

### Knowledge
Sports | Leadership & People | Tech & Engineering | Audiobooks | Misc -- all in sections. Learning plan with checkbox progress.

### Links
`knowledge/links.json`. Bookmark collection.

### Network
Automatic contact list from direct mails and small-group meetings (max 8 attendees).

| Sort | Display |
|---|---|
| Frequent | Top 20 (Inner Circle, gold star), tree view by department |
| A-Z | Alphabetical, flat |
| Department | Tree view, groups collapsed |

- Search field with real-time filter. Manual add via `+ Contact`.
- Click opens modal (edit role, delete contact).
- Frequency counts never displayed (privacy).

**Contact Extraction** (in `export_outlook_mails.ps1`):
- Received mails: only when sole To-recipient (prio `direct`/`chef`) -> mailFreq++
- Sent mails: only 1:1 (exactly one recipient) -> mailFreq++
- Calendar: only meetings with max 8 ATTENDEEs -> meetFreq++
- Filter: own names, DL/SAP/Cloud/External prefixes (also suffix and comma variants), `_`/digits
- Department lookup via Outlook COM (cached). `rolle` never overwritten.
- Cleanup: contacts without mailFreq and without meetFreq are removed.

Data format:
```json
[{"name":"Daniel K.","rolle":"Tech Lead","first":"13.01.","last":"22.04.","mailFreq":12,"meetFreq":3,"freq":15,"_weekId":"2026-W17"}]
```

### Jira
- Two trees: **"Meine"** (personally assigned items, grouped by project, open by default) + **"SL Toolset for Cloud"** (toolset items, grouped by release, collapsed by default).
- All items as compact pill buttons color-coded by status. "Meine" shows shortened Jira key (CLMSLORCHESTRATOR -> SLOCON, CLMSLCCI4ABAP -> CLOUDLM), toolset shows category-number (e.g. "RAMP-2859"). Summary + status as tooltip.
- Toolset items have `releases` and `category` fields in `data/jira.json`. Categories: SLV (HECSPCVAL), REGR (CLMOQHEC single-release), RAMP (CLMCONSUMABILITY / CLMOQHEC multi-release).
- Items with multiple releases appear under each release.
- Releases sorted descending (newest on top), categories: REGR > RAMP > SLV. Items within sorted by status: Running/In Progress > On Hold > Open/To Do > Planned.
- Status colors: In Progress/Running (blue), Blocked (red), On Hold (orange), Open/Planned/To Do (gray), Done (green). No priority colors.
- Click on pill opens Jira in system browser (`openExternal()` via `/open-url`).
- Smart Sync/Setup button: polls `jira_status.json`, switches to setup on token error.
- Last sync timestamp shown below (always date + time).
- Auth: OAuth 2.0 via SAP MCP proxy.

### News
- Positive news from squirrel-news.net, summarized via Claude API into 3-5 emoji bullets.
- Auto-trigger on app start when outdated. Refresh button in footer.
- Timestamp below (same format as Jira).

### Goals
- Annual goals from `knowledge/goals.json` with automatic mail matching.
- 4 collapsible sections (one per goal) with weight, description and matched mails of the week.
- Keyword matching: each goal has a `keywords` array, matched against `mailData[].subject` (case-insensitive).
- Header shows year, calendar week and total mail match count.
- Meta line: timestamp of last calculation.
- Refresh button in footer: reloads mail and goals data, recalculates matches immediately. Once daily is sufficient.
- Per goal: weight badge (gold), title, mail count, up to 15 mails with priority marker, sender, date and keyword badges. Click on mail expands body.

## 8. Write Server

Node.js on `127.0.0.1:9001`. PowerShell path hardcoded (`C:\Windows\System32\WindowsPowerShell\v1.0\powershell.exe`). Spawn without `detached`/`unref` (corporate Windows workaround).

| Method | Path | Action |
|---|---|---|
| GET | `/ping` | Health check |
| POST | `/modal-lock?state=1\|0` | Write/delete lock file |
| POST | `/run-export-mails` | Spawn mail export |
| POST | `/run-summarize` | Spawn AI summary |
| POST | `/run-export-calendar` | Spawn calendar export |
| POST | `/run-plan-week` | Day rating (execFile, 8 min timeout) |
| POST | `/run-generate-quote[?force=1]` | Spawn daily quote |
| POST | `/run-generate-news[?force=1]` | Spawn daily news |
| POST | `/run-sync-jira` | Jira sync (execFile, 60s) |
| POST | `/run-setup-jira` | Jira setup (execFile, 210s) |
| POST | `/open-url` | Open URL in system browser (protocol check) |
| PUT | `/{file}` | Write file (allowlist) |

**PUT Allowlist:** `data/` -> actions, notes, contacts, jira, jira_status. `knowledge/` -> links, learningplan_progress, sport, quotes, news.

## 9. PowerShell Scripts

### secrets.ps1
Not committed. Contains `$ApiKey`, `$ProxyUri`, `$MyAddresses`, `$MyDisplayNames`, `$ChefDept`, `$MitarbeiterDept`, `$Mitarbeiter`.

### export_outlook_today.ps1
Calendar today + tomorrow as ICS. Free -> TRANSPARENT, Tentative -> TENTATIVE, Optional -> OPT-PARTICIPANT. ATTENDEEs for network extraction.

### export_outlook_mails.ps1
Inbox + Sent (Monday to today). Writes `mails_today.json`, `mail_sync_status.json`, `config.json`, `contacts.json`. Contact extraction with department lookup and race-condition protection.

### export_sync_prep.ps1
Tomorrow's 1:1 syncs -> `sync_files.json` + `KW{n}-{Name}.json`. Event detection via name matching. Mail assignment exclusively by email address (no name matching, prevents false matches on shared first names). Sent mails: only direct To-recipients (no CC), Exchange addresses resolved.

### summarize_mails.ps1
Claude API via corporate proxy -> `summary_KW{n}.json`.

### plan_week.ps1
Daily at 6 PM + on-demand. 5 separate API calls (MON-FRI). Emoji + one sentence per day. Skips when `modal.lock` exists.

### generate_quote.ps1
Quote from classical German literature via Claude API. `-force` for regeneration.

### generate_news.ps1
squirrel-news.net -> Claude API -> 3-5 emoji bullets. JSON built manually (ConvertTo-Json bug). `-force` for regeneration.

### sync_jira.ps1
Two JQL queries: (1) personal items (`assignee = user`), (2) toolset items (`project in (CLMCONSUMABILITY, CLMOQHEC, HECSPCVAL)` with `fixVersion`). Release numbers extracted from `fix_versions` (e.g. "SL Toolset for Cloud 2604 (T2026.15)" -> "2604"). Category auto-derived: HECSPCVAL -> SLV, CLMOQHEC single-release -> REGR, CLMOQHEC multi-release / CLMCONSUMABILITY -> RAMP. Manually set `category` is preserved. OAuth token refresh. UTF-8 without BOM. Manual JSON build (ConvertTo-Json bug).

### setup_jira.ps1
PKCE flow with browser login (SAP SSO). Callback on port 41562. Status messages in `jira_status.json`.

### setup.ps1
One-time setup: checks prerequisites, registers 11 scheduled tasks, creates shortcuts.

| Task | Schedule |
|---|---|
| Write Server / Live Server / Dashboard | on logon |
| Mail Export | 07:30 + 12:30 |
| Sync Prep | 17:00 |
| Calendar Export | 06:00 + 12:00 |
| Day Planning | 18:00 |
| Jira Sync | 08:00 + 13:00 |

## 12. Security

- **Secrets**: Only in `secrets.ps1` (not committed). `.gitignore` with `# PRIVACY`.
- **XSS**: All user data via `escapeHtml()` (incl. `"` for attribute contexts). `safeHref()` blocks `javascript:` URLs. `linkify()` uses `safeHref()` + `rel="noopener noreferrer"`.
- **Write Server**: Allowlist for PUT, localhost only, CORS restricted to `127.0.0.1:5500`, 10 MB body limit.
- **URL Opening**: `/open-url` checks protocol (`http:`/`https:`) via `new URL()`, opens via `explorer.exe`.
- **Names**: `shortName()` (first name + first letter of last name). Frequency counts never displayed.
- **Calendar**: No inline `onclick` with user data -- `data-*` attributes + `addEventListener`.

## 14. Known Pitfalls

| Topic | Details |
|---|---|
| No `detached`/`unref` | Causes silent spawn failures on corporate Windows. Use `{ stdio: 'ignore' }` instead. |
| PowerShell ASCII only | Em-dashes, umlauts cause silent parser errors on non-interactive start. |
| CRLF required | PS scripts need Windows line endings. `unix2dos` after writing. |
| PS path hardcoded | `SystemRoot` is empty when Node starts from Git Bash. |
| Outlook COM + `/it` | Requires running Outlook + interactive session. GPO prevents `Register-ScheduledTask`. |
| `ConvertTo-Json` bug | Arrays in hashtables are serialized as `{value:[],Count:N}`. Build JSON manually. |
| `modal.lock` in `C:\Temp` | Not in `data/` -- live-server would trigger reload. |
| UTF-8 without BOM | `[System.Text.UTF8Encoding]::new($false)` for all JSON writes. |
| `live-server --ignore` | `data,knowledge` excluded from watch. |
| ISOWeek in PS 5.1 | `GetWeekOfYear` instead of `[System.Globalization.ISOWeek]` (requires .NET 5+). |
